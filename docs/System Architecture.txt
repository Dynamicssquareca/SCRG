1. HIGH-LEVEL ARCHITECTURE
--------------------------------------------------------------------------------

The system follows a classic 3-Tier Web Architecture:

  ┌─────────────────────────────────────────────────────────────────────┐
  │                        PRESENTATION TIER                           │
  │                   (React.js SPA — Browser)                         │
  │                                                                     │
  │  ┌──────────┐  ┌───────────┐  ┌──────────┐  ┌──────────────────┐  │
  │  │  Upload   │  │  Reports  │  │  Client  │  │  Login / Auth    │  │
  │  │  Page     │  │  Listing  │  │  Master  │  │  Page            │  │
  │  └──────────┘  └───────────┘  └──────────┘  └──────────────────┘  │
  └──────────────────────────┬──────────────────────────────────────────┘
                             │ HTTPS (REST API)
                             ▼
  ┌─────────────────────────────────────────────────────────────────────┐
  │                         APPLICATION TIER                           │
  │                    (Node.js + Express.js)                           │
  │                                                                     │
  │  ┌──────────────┐  ┌───────────────┐  ┌─────────────────────────┐ │
  │  │  Auth        │  │  Upload &     │  │  Report Generation     │ │
  │  │  Middleware   │  │  Validation   │  │  Engine (ExcelJS)      │ │
  │  │  (JWT)       │  │  Service      │  │                         │ │
  │  └──────────────┘  └───────────────┘  └─────────────────────────┘ │
  │  ┌──────────────┐  ┌───────────────┐  ┌─────────────────────────┐ │
  │  │  Data        │  │  Client       │  │  File Storage           │ │
  │  │  Processing  │  │  Master       │  │  Service                │ │
  │  │  Service     │  │  Service      │  │  (Local / S3)           │ │
  │  └──────────────┘  └───────────────┘  └─────────────────────────┘ │
  └──────────────────────────┬──────────────────────────────────────────┘
                             │ Database Queries
                             ▼
  ┌─────────────────────────────────────────────────────────────────────┐
  │                          DATA TIER                                 │
  │                    (PostgreSQL Database)                            │
  │                                                                     │
  │  ┌──────────┐  ┌───────────┐  ┌──────────┐  ┌──────────────────┐  │
  │  │  users   │  │  clients  │  │  uploads  │  │  cases           │  │
  │  └──────────┘  └───────────┘  └──────────┘  └──────────────────┘  │
  │  ┌──────────┐  ┌───────────┐                                       │
  │  │  reports │  │  report_  │                                       │
  │  │          │  │  files    │                                       │
  │  └──────────┘  └───────────┘                                       │
  └─────────────────────────────────────────────────────────────────────┘


2. COMPONENT ARCHITECTURE
--------------------------------------------------------------------------------

┌──────────────────────────────────────────────────────────────────────────┐
│                         REQUEST FLOW                                    │
│                                                                          │
│  User uploads Excel ──► Multer (file upload middleware)                  │
│       │                                                                  │
│       ▼                                                                  │
│  Validation Service ──► Checks column schema, data types, formats       │
│       │                                                                  │
│       ▼                                                                  │
│  Data Processing Service ──► Parses Excel using xlsx/exceljs            │
│       │                      Normalizes dates, trims strings            │
│       │                      Groups data by Client + Month              │
│       │                                                                  │
│       ▼                                                                  │
│  Database Insertion ──► Stores raw cases in `cases` table               │
│       │                 Stores upload metadata in `uploads` table        │
│       │                                                                  │
│       ▼                                                                  │
│  Report Generation Engine                                                │
│       │   For each unique client:                                        │
│       │     1. Query cases (open vs resolved)                            │
│       │     2. Query client master (hours, manager, CSM)                 │
│       │     3. Compute ticket summary & balance hours                    │
│       │     4. Build Excel workbook (ExcelJS) with 3 sheets             │
│       │     5. Apply template styling (colors, fonts, widths)            │
│       │     6. Save .xlsx to file storage                                │
│       │     7. Record in `reports` / `report_files` tables              │
│       │                                                                  │
│       ▼                                                                  │
│  Response ──► Return list of generated reports with download URLs        │
└──────────────────────────────────────────────────────────────────────────┘

4. SECURITY ARCHITECTURE
--------------------------------------------------------------------------------

  ┌─────────────────────────────────────────────────────────────────┐
  │                    SECURITY LAYERS                              │
  │                                                                 │
  │  Layer 1: HTTPS / TLS 1.3                                      │
  │     All traffic encrypted in transit                            │
  │                                                                 │
  │  Layer 2: Authentication (JWT)                                  │
  │     Login → issue access token (15 min) + refresh token (7d)   │
  │     All API routes protected via auth middleware                │
  │                                                                 │
  │  Layer 3: Authorization (Role-Based)                            │
  │     Admin    → full access (CRUD clients, upload, reports)     │
  │     Operator → upload data, generate & download reports        │
  │     Viewer   → browse & download reports only                  │
  │                                                                 │
  │  Layer 4: Input Validation                                      │
  │     File type check (xlsx/xls/csv only)                        │
  │     File size limit (50 MB)                                     │
  │     Column schema validation                                    │
  │     SQL injection prevention via parameterized queries (Knex)  │
  │                                                                 │
  │  Layer 5: Data at Rest                                          │
  │     Passwords hashed with bcrypt (12 rounds)                   │
  │     Database backups encrypted                                  │
  │     Generated files stored in restricted directory              │
  └─────────────────────────────────────────────────────────────────┘


5. SCALABILITY CONSIDERATIONS
--------------------------------------------------------------------------------
- Stateless API → can horizontally scale Node.js instances behind a load
  balancer.
- Report generation can be offloaded to a background job queue (Bull/Redis)
  for large datasets.
- File storage can migrate from local disk to S3/Azure Blob for durability.
- Database connection pooling via pg-pool (max 20 connections).
- React SPA served from CDN for fast global delivery.
